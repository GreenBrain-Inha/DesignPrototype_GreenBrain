# GreenBrain — Design Prototype

AI 챗봇 탄소 책임 서비스 **GreenBrain**의 **프론트엔드 디자인 프로토타입**입니다.
백엔드 없이 UI/UX 흐름만 확인하기 위한 목적으로 제작되었습니다.

## 화면 구성

| 화면 | URL | 파일 |
|------|-----|------|
| 로그인 | `/login` | `src/app/pages/Login.tsx` |
| 회원가입 | `/signup` | `src/app/pages/Signup.tsx` |
| 온보딩 | `/onboarding` | `src/app/pages/Onboarding.tsx` |
| 채팅 | `/chat` | `src/app/pages/Chat.tsx` |
| 챌린지 | `/challenges` | `src/app/pages/Challenges.tsx` |
| 인증 피드 | `/challenges/feed` | `src/app/pages/ChallengeFeed.tsx` |

## 실행 방법

```bash
pnpm install
pnpm build
```

## 디자인 프로토타입에서 처리할 항목

> 프로토타입의 목적은 핵심 UX 루프와 화면 흐름 검증입니다.
> 아래 항목은 백엔드 없이도 수정 가능한 항목입니다.

| 중요도 | 항목 | 현재 상태 | 수정 방향 | 관련 파일 |
|:------:|------|-----------|-----------|-----------|
| 🟡 권장 | **온보딩 데이터 → 챌린지 미연결** | 온보딩에서 선택한 교통수단·식단·주거가 챌린지 추천에 반영되지 않아 개인화 UX 흐름 검증 불가 | 온보딩 응답을 임시 저장(localStorage 등)해 챌린지 선택 로직에 반영 | `Onboarding.tsx`, `Challenges.tsx:42` |
| 🟡 권장 | **진행 중 챌린지 중복 화면 없음** | 이미 챌린지를 수락한 상태에서 `/challenges`에 재진입하면 새 챌린지가 다시 랜덤 표시됨 | 진행 중 챌린지가 있을 때 기존 챌린지를 그대로 보여주는 화면 분기 추가 | `Challenges.tsx:42` |
| 🟡 권장 | **토큰 색상 임계값 UI 검증** | 토큰 색상이 50% / 20% 기준으로 변하지만 PRD에 임계값이 명시되어 있지 않아 디자인 검토 필요 | 실제 사용성 테스트 후 임계값(%) 확정 | `Chat.tsx:40` |

## 실제 구현 시 처리할 항목

> 백엔드 연동이 필요하거나 인프라 의사결정이 필요한 항목입니다.
> 프로토타입에서는 mock으로 대체된 상태이며, 실제 구현 시 반드시 처리해야 합니다.

| 분류 | 항목 | 내용 | 관련 PRD |
|------|------|------|----------|
| **기술 스택** | 프레임워크 교체 | Vite+React → Next.js 마이그레이션 | 기술 스택 |
| **기술 스택** | 백엔드 구축 | FastAPI + Supabase + PostgreSQL 연동 | 기술 스택 |
| **채팅** | AI 실제 연동 | GPT-4o mini API 호출 (현재 하드코딩 응답) | F-C01 |
| **채팅** | 탄소 계산 연동 | ecologits로 실제 gCO₂eq 계산 (현재 `Math.random()`) | F-C02 |
| **채팅** | ecologits 오류 처리 | 실패 시 "탄소 계산 불가" 표시, 해당 메시지 토큰 미차감 | F-C02 |
| **채팅** | 일일 토큰 초기화 | KST 자정 기준 150 토큰으로 서버 측 초기화 | F-C06 |
| **채팅** | 좋아요 보상 일일 상한 | 하루 최대 50토큰 서버 측 집계·제한 | F-C07 |
| **인증** | JWT 발급·관리 | HttpOnly 쿠키, 만료 7일, 갱신 처리 | F-A03 |
| **인증** | 이메일 인증 발송 *(고도화)* | Supabase Auth 기반 인증 링크 발송 (24시간 유효) — MVP에서는 생략, 추후 고도화 단계에서 적용 | F-A02 |
| **인증** | Rate limiting | 로그인 5회 초과 → 15분 잠금 | F-A03 |
| **챌린지** | 자동 생성 로직 | 생활 습관 프로필 + 완료 이력 기반 챌린지 생성 | F-CH01 |
| **챌린지** | 난이도 점진 상승 | 누적 완료 횟수에 따른 난이도 자동 조정 | F-CH03 |
| **챌린지** | 일일 3개 상한 | 사용자당 하루 챌린지 생성 횟수 서버 측 제한 | F-CH06 |
| **사진·피드** | 서버 사이드 리사이즈 | 업로드 사진 최대 1280px 리사이즈 후 저장 | F-V01 |
| **사진·피드** | 스토리지 전환 | MVP: 로컬 파일시스템 → 프로덕션: S3 호환 스토리지 | 2.5 |
| **어뷰징 방지** | 본인 사진 좋아요 차단 | 서버 측 업로더 ID와 요청자 ID 비교 | F-V03 |
| **어뷰징 방지** | 신규 계정 좋아요 차단 | 계정 생성 후 24시간 이내 좋아요 버튼 비활성화 | F-V03 |
| **어뷰징 방지** | 중복 좋아요 방지 | DB 유니크 제약으로 동일 사용자 중복 좋아요 차단 | F-V03 |
| **데이터** | 영속성 확보 | 모든 상태(토큰·챌린지·좋아요)를 PostgreSQL에서 관리 | 2.5 |
