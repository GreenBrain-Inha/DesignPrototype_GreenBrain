# PRD: GreenBrain 프론트엔드 (MVP)

> 작성일: 2026-05-04 | 팀: GreenBrain

## 1. 역할 경계

프론트엔드는 **UI 렌더링, 사용자 인터랙션, 클라이언트 유효성 검사, API 호출**을 담당한다.
비즈니스 로직, 데이터 저장, 보안 정책은 백엔드 책임이며 프론트엔드는 백엔드 응답을 신뢰한다.

| 책임 | 프론트엔드 | 백엔드 |
|------|:---------:|:------:|
| 화면 렌더링 | ✅ | |
| 클라이언트 유효성 검사 (형식) | ✅ | |
| API 호출 및 응답 처리 | ✅ | |
| 로딩 / 에러 / 빈 상태 표시 | ✅ | |
| JWT 토큰 저장 (HttpOnly 쿠키) | | ✅ |
| 탄소 계산 (ecologits) | | ✅ |
| AI 응답 생성 (GPT-4o mini) | | ✅ |
| 토큰 차감 / 회복 계산 | | ✅ |
| 챌린지 자동 생성 | | ✅ |
| 좋아요 보상 지급 | | ✅ |
| 어뷰징 방지 | | ✅ |
| 파일 저장 / 리사이즈 | | ✅ |
| 일일 토큰 초기화 | | ✅ |

## 4. 화면별 UI 요구사항

### 4.1 로그인 (`/login`)

**UI 구성**
- 로고 + 서비스 설명
- 이메일 입력 필드
- 비밀번호 입력 필드
- 로그인 버튼
- 회원가입 링크

**클라이언트 유효성 검사**
- 이메일 형식 확인
- 비밀번호 미입력 방지

**API 연동**

```
POST /api/auth/login
Body: { email, password }
성공 → /chat 이동 (쿠키는 백엔드가 Set-Cookie로 설정)
실패 → 에러 메시지 표시 ("이메일 또는 비밀번호가 올바르지 않습니다")
```

**상태 처리**
- 로딩: 버튼 비활성화 + 스피너
- 에러: 폼 하단 인라인 에러 메시지

---

### 4.2 회원가입 (`/signup`)

**UI 구성**
- 이메일 입력 필드
- 비밀번호 입력 필드
- 비밀번호 확인 입력 필드
- 가입하기 버튼
- 로그인 링크

**클라이언트 유효성 검사**
- 이메일 형식 확인
- 비밀번호: 최소 8자, 대소문자 + 숫자 혼합 (정규식)
- 비밀번호 일치 여부

**API 연동**

```
POST /api/auth/signup
Body: { email, password }
성공 → /onboarding 이동
실패 → 에러 메시지 표시 ("이미 사용 중인 이메일입니다" 등)
```

**상태 처리**
- 로딩: 버튼 비활성화 + 스피너
- 에러: 폼 하단 인라인 에러 메시지

---

### 4.3 온보딩 (`/onboarding`)

**UI 구성**
- 단계 표시 (1/3, 2/3, 3/3) + 프로그레스바
- 각 단계별 질문 + 선택지 카드 (아이콘 + 라벨)
- 이전 / 다음 버튼

**질문 구성**

| 단계 | 질문 | 선택지 |
|------|------|--------|
| 1 | 주로 사용하는 교통수단 | 대중교통, 자가용, 자전거, 도보 |
| 2 | 식단 유형 | 비건, 채식 위주, 균형 식단, 육식 위주 |
| 3 | 주거 형태 | 아파트, 단독주택, 원룸/오피스텔, 셰어하우스 |

**인터랙션**
- 선택된 카드: 초록 테두리 + 배경 + 체크 아이콘 표시
- 선택 전 다음 버튼 비활성화
- 3단계 완료 후 저장 API 호출

**API 연동**

```
POST /api/user/onboarding
Body: { transportation, diet, housing }
성공 → /chat 이동
실패 → 에러 토스트 표시
```

**상태 처리**
- 로딩: 마지막 단계 버튼 스피너

---

### 4.4 채팅 (`/chat`)

**UI 구성**
- 헤더: 로고 + 탄소 토큰 표시 (숫자 + 프로그레스바) + 네비게이션 메뉴
- 메시지 목록 (사용자: 오른쪽 초록 말풍선 / AI: 왼쪽 흰색 말풍선)
- AI 메시지 아래 탄소 배출 정보 카드 (gCO₂eq 수치 + 비유 텍스트)
- 하단 입력창 + 전송 버튼

**토큰 프로그레스바 색상**

| 잔여 비율 | 색상 |
|-----------|------|
| 50% 초과 | 초록 (green-500) |
| 20% ~ 50% | 앰버 (amber-500) |
| 20% 미만 | 빨강 (red-500) |

**API 연동**

```
// 페이지 진입 시 현재 토큰 조회
GET /api/user/tokens
응답: { remaining: number, max: 150 }

// 메시지 전송
POST /api/chat/message
Body: { message: string }
응답: { reply: string, carbon_cost: number, remaining_tokens: number }
```

**상태 처리**
- 메시지 전송 중: 말풍선 로딩 애니메이션 (... 점 세 개)
- ecologits 실패 시: 탄소 정보 카드에 "탄소 계산 불가" 표시 (토큰 차감 없음 — 백엔드 판단)
- 토큰 소진 시: 입력창 비활성화 → 토큰 소진 안내 화면 전환 → `/challenges`로 이동 버튼

**토큰 소진 화면** (토큰 0 상태)
- 경고 아이콘 + "오늘의 탄소 토큰을 모두 사용했습니다" 메시지
- "챌린지 시작하기" 버튼 → `/challenges` 이동

---

### 4.5 챌린지 (`/challenges`)

**UI 구성 — 상태별 분기**

**① 챌린지 수락 전**
- 챌린지 카테고리 뱃지 + 아이콘 + 제목 + 설명
- 보상 안내 (업로드 시 +20 토큰 / 좋아요 3개당 +20 토큰)
- "챌린지 수락하기" 버튼

**② 챌린지 수락 후 (사진 업로드 단계)**
- 챌린지 정보 요약
- 사진 업로드 영역 (드래그 앤 드롭 또는 파일 선택)
- 선택된 사진 미리보기 + 다시 선택 버튼
- "인증 사진 업로드" 버튼

**③ 일일 상한 도달 (챌린지 3개 완료)**
- "오늘의 챌린지를 모두 완료했습니다" 안내
- 피드 이동 버튼

**파일 클라이언트 유효성 검사**
- 허용 포맷: JPEG, PNG, WebP
- 최대 크기: 10MB
- 위반 시 인라인 에러 메시지 표시 (alert 사용 금지)

**API 연동**

```
// 현재 챌린지 조회 (진입 시)
GET /api/challenges/current
응답: { challenge: Challenge | null, daily_count: number }

// 챌린지 수락
POST /api/challenges/{id}/accept
응답: { challenge: Challenge }

// 사진 업로드
POST /api/challenges/{id}/verify
Body: FormData { photo: File }
응답: { tokens_rewarded: number, remaining_tokens: number }
성공 → 토큰 업데이트 후 /chat 이동
```

**상태 처리**
- 조회 중: 스켈레톤 카드
- 업로드 중: 버튼 비활성화 + 스피너
- 에러: 인라인 에러 메시지

---

### 4.6 인증 피드 (`/challenges/feed`)

**UI 구성**
- 헤더: "챌린지 피드" + 네비게이션 메뉴
- 커뮤니티 배너 (좋아요 안내)
- 피드 카드 목록 (최신순)
  - 업로더 아바타 + 닉네임 + 업로드 시각
  - 챌린지 제목 + 아이콘
  - 인증 사진
  - 좋아요 버튼 + 좋아요 수
- 빈 상태: "아직 인증된 챌린지가 없습니다" + 챌린지 시작 버튼

**좋아요 버튼 상태**

| 조건 | 버튼 상태 |
|------|-----------|
| 기본 | 활성화, 비선택 |
| 이미 좋아요한 사진 | 비활성화, 초록 강조 |
| 본인이 업로드한 사진 | 비활성화, 회색 |

**API 연동**

```
// 피드 목록 조회
GET /api/feed?page=1
응답: { items: FeedItem[], has_next: boolean }

// 좋아요
POST /api/feed/{item_id}/like
응답: { likes: number, liked_by_me: boolean }
실패(중복/본인/신규계정) → 에러 메시지 표시
```

**상태 처리**
- 조회 중: 스켈레톤 카드 3개
- 좋아요 요청 중: 버튼 즉시 낙관적 업데이트(Optimistic UI) → 실패 시 롤백
- 빈 상태: 안내 메시지 + 챌린지 이동 버튼

## 6. 전역 상태 관리

React Context로 관리할 전역 상태:

```ts
interface AppState {
  user: {
    id: string;
    email: string;
    onboarding_complete: boolean;
  } | null;
  tokens: {
    remaining: number;   // 백엔드 응답 기준
    max: number;         // 150 고정
  };
}
```

- **토큰**: API 응답마다 `remaining_tokens`로 동기화. 클라이언트에서 자체 계산하지 않음.
- **사용자 정보**: 로그인 성공 시 설정, 로그아웃 시 초기화.

## 7. API 에러 공통 처리

| HTTP 상태 | 처리 |
|-----------|------|
| 401 | 로그인 페이지로 리다이렉트 |
| 403 | 인라인 에러 메시지 표시 |
| 422 | 필드별 유효성 에러 메시지 표시 |
| 500 | "일시적인 오류가 발생했습니다" 토스트 |
| 네트워크 오류 | "인터넷 연결을 확인해주세요" 토스트 |
